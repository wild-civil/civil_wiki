# 屏幕接口与带宽估算

主控（MCU / GPU）要把一帧帧像素数据送到屏驱 IC。不同接口就是不同的「数据管道」：管道越粗、跑得越快，能撑的分辨率与刷新率就越高。这一篇讲清常见接口和带宽怎么算。

## 一、常见接口一览

### 1. RGB 并行（RGB TTL / MCU 8080）

- **并行总线**：R/G/B 各数据线 + 行同步 HSYNC + 场同步 VSYNC + 像素时钟 PCLK，一起并排走线。
- 简单、MCU 友好，但引脚多（24 位色就要 ~28 根线）、走线易受干扰、传输距离短。
- 适合小屏（< 7"）、MCU 直驱；常说的「RGB565 屏」多半走这种接口。

### 2. MIPI DSI

- 手机 / 平板标配。串行差分（1~4 条 lane），引脚少、速率高、省电。
- 自带协议层（命令模式 / 视频模式），物理层用 D-PHY。

### 3. LVDS

- 老式笔记本、工控屏常用。低压差分串行，抗干扰强、传输距离远。
- 用一对或两对差分线传整帧数据。

### 4. eDP（Embedded DisplayPort）

- 笔记本内置屏主流。基于 DisplayPort，串行、带宽高，轻松吃下高分辨率高刷。

### 5. HDMI / DisplayPort

- 对外视频输出接口（接显示器/电视），不是「屏内」接口，但常被拿来对比带宽。

## 二、带宽怎么算

核心公式：

$$
\text{带宽} \approx \text{像素数} \times \text{色深(bpp)} \times \text{刷新率(Hz)} \times (1 + \text{消隐比})
$$

**消隐（blanking）**：每行、每帧之间都有回扫空隙（HSYNC/VSYNC 期间不传有效像素），约占 20%~30%，所以整体再乘 **1.2 ~ 1.3**。

示例：1080p（1920×1080）、RGB888（24 bpp）、60 Hz

- 像素数 = 2,073,600
- 裸带宽 = 2,073,600 × 24 × 60 ≈ **2.99 Gbit/s ≈ 373 MB/s**
- 含消隐（×1.25）≈ **3.74 Gbit/s**

同样分辨率同样刷新率，换成 16 bpp（RGB565）就只有约 **249 MB/s**——这就是为什么嵌入式爱用 RGB565（详见[屏幕色彩格式与色深](屏幕色彩格式与色深.md)）。

## 三、接口带宽对照（大致上限）

| 接口 | 典型带宽 | 常用场景 |
|---|---|---|
| RGB TTL | < 1 Gbit/s | MCU 小屏 |
| MCU 8080 | 很低 | 很低分辨率 |
| LVDS | ~3–7 Gbit/s | 工控 / 老笔记本 |
| MIPI DSI（4 lane） | ~10+ Gbit/s | 手机 / 平板 |
| eDP 1.4 | ~25.9 Gbit/s | 笔记本高刷 |
| HDMI 2.0 / 2.1 | 18 / 48 Gbit/s | 外接显示器 |

## 四、选型速记

- MCU + 小屏 → **RGB 并行** 或 SPI
- 手机 / 便携 → **MIPI DSI**
- 笔记本内置 → **eDP**
- 工控长距离 → **LVDS**
- 对外输出 → **HDMI / DP**

> 想验证某块屏能不能跑满目标分辨率，用上面的公式算出的带宽，必须 ≤ 接口上限。分辨率/刷新率/PPI 之间的关系，见[分辨率、刷新率与像素密度（PPI）](分辨率_刷新率与PPI.md)。
