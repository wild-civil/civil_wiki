# HDR 与色彩空间（sRGB / P3 / Display-P3）

前两篇讲了「颜色用多少位表示」「屏幕怎么发光」。这一篇补上「颜色的范围有多大、亮到什么程度」——也就是色彩空间与 HDR。这是选购手机/显示器时「色域」「HDR600」这些词背后的含义。

## 一、色彩空间：颜色的范围

同一个「红色」，在不同标准下数值不同。**色彩空间**定义了两件事：能显示哪些颜色（**色域**），以及这些颜色怎么编码。

常见色彩空间：

| 空间 | 特点 | 典型场景 |
|---|---|---|
| **sRGB** | 最通用，网络/Windows 默认，色域较小 | 网页、日常 |
| **Adobe RGB** | 印刷向，绿青更广 | 摄影后期 |
| **DCI-P3** | 数字影院，红绿比 sRGB 广（电影感） | 影视 |
| **Display-P3** | 苹果推广的 P3 + sRGB 伽马 | 手机/Mac 屏幕 |

> 注意：「支持 P3」≠ 出厂校过色。一块广色域屏如果**不做色彩管理**，显示 sRGB 内容会过饱和、颜色发「艳」失真。广色域的价值在于「需要时能显示更宽的色」，而不是永远拉满。

## 二、亮度单位：nit

亮度用 **cd/m²（尼特, nit）** 表示。普通屏约 250–350 nit；HDR 通常要求峰值 **600–1000+ nit**。高亮是 HDR「通透感」的关键——光够亮，高光细节才压得住。

## 三、什么是 HDR

- **SDR（标准动态范围）**：亮度区间窄、多为 8-bit。
- **HDR（高动态范围）**：更亮的高光 + 更暗的暗部 + 更广色域 + 10-bit 色深，画面更立体。

但要实现「真 HDR」，需**同时满足**：面板峰值亮度高 + 黑位低 + 10-bit 色深 + 广色域 + 片源带 HDR 元数据。少一条就是「假 HDR」。

> 色深正是[屏幕色彩格式与色深](屏幕色彩格式与色深.md)里讲的：HDR 至少要 10-bit，8-bit 屏硬上 HDR 内容会出现明显色带。

## 四、HDR 标准对照

| 标准 | 位深 | 元数据 | 备注 |
|---|---|---|---|
| **HDR10** | 10-bit | 静态 | 开放、主流 |
| **HDR10+** | 10-bit | 动态 | 三星主导 |
| **Dolby Vision** | 12-bit | 动态 | 需授权，效果佳 |
| **HLG** | 10-bit | 广播向 | 广电直播 |

**动态元数据** = 逐场景甚至逐帧调整亮度映射，比静态元数据更精准，这也是 Dolby Vision / HDR10+ 观感更稳的原因。

## 五、把这一章串起来

选屏其实是一个「木桶」：

- **色深**（第一篇）决定 HDR 至少 10-bit；
- **面板**（第二篇：LCD/OLED/Mini-LED）决定峰值亮度与黑位能否撑起 HDR；
- **接口带宽**（第三篇）决定 4K HDR 高刷能否传得动；
- **分辨率/PPI**（第四篇）决定细腻程度；
- **色彩空间 + HDR**（本篇）决定颜色范围与明暗层次。

五者配套，才不会有一块短板拖后腿。

> 想从零搭一套显示方案（MCU 小屏还是电脑高刷屏），建议按「面板 → 接口 → 色深/色彩空间 → 分辨率/PPI」的顺序逐项确认。
