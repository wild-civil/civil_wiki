## 1. 核心规格速查

| 参数 | 规格 |
|------|------|
| **CPU 内核** | 青稞 V4 32位 RISC-V（QingKe V4） |
| **最高主频** | 80 MHz |
| **指令集** | RV32IMACF + 自研字节/半字压缩指令 |
| **程序存储** | 448 KB Flash / 512 KB（不同批次/型号有差异） |
| **数据存储** | 32 KB SRAM |
| **工作电压** | 2.3 V ~ 3.6 V |
| **睡眠功耗** | 低至 0.3 μA |
| **封装** | QFN-48-EP (5 mm × 5 mm) |

### CH582 vs CH583 差异

| 特性 | CH582 | CH583 |
|------|-------|-------|
| SPI 主机数量 | 1 个 | 2 个 |
| 最低工作电压 | 2.3 V | 1.75 V |
| 其他 | 基础型号 | 增强型号，外设更全 |

> 注：两者引脚兼容，评估板例程基本通用，选择时主要看外设需求。

## 2. 无线通信

### BLE 5.3

- 集成 **2 Mbps 低功耗蓝牙** 协议栈（物理层 + 链路层 + Host）
- 支持角色：
  - **Peripheral（从机）**：最常用，如传感器、键鼠、手环
  - **Central（主机）**：扫描、连接多个从机
  - **Observer（观察者）**：仅扫描，不连接
  - **Broadcaster（广播者）**：仅广播，不接收
- 支持 **BLE MESH** 组网（需配合例程）
- 支持 **2.4G 私有协议** 模式（非标准 BLE，适合自定义遥控器）

### 实际应用例程（官方 EVT 包）

| 例程 | 说明 |
|------|------|
| `Peripheral` | 通用从机，含 5 种属性特征（读/写/通知/可读可写/安全可读） |
| `HID_Keyboard` | 蓝牙键盘 |
| `HID_Mouse` | 蓝牙鼠标 |
| `BLE_UART` | 蓝牙串口透传（最常用） |
| `HeartRate` | 心率计（模拟 Health 设备） |
| `MultiCentral` | 主机多连接（同时连 3 个从机） |
| `MESH` 系列 | 天猫精灵灯、厂商自定义模型、朋友节点/低功耗节点等 |
| `OTA` 系列 | 无线固件升级（备份升级 / 固定库升级） |

## 3. USB 外设

- **2 组独立全速 USB 2.0（12 Mbps）**
  - 可同时做 **Host + Device**
  - 或双 Device / 双 Host 组合
- 常见用法：
  - USB-CDC（虚拟串口）用于调试/升级
  - USB-HID 用于键鼠设备
  - USB-U盘 用于数据记录

## 4. 其他片上外设

| 外设 | 数量/规格 | 学习要点 |
|------|-----------|----------|
| **UART** | 4 组 | 蓝牙透传、日志打印、GPS 等 |
| **SPI** | 2 组（CH583）/ 1 组（CH582） | 驱动屏幕、Flash、传感器 |
| **ADC** | 14 通道，12 bit | 电池电压、模拟传感器采集 |
| **触摸按键** | 内置检测模块 | 省外部电容，适合做面板 |
| **RTC** | 内置日历 | 低功耗定时唤醒、时间戳 |
| **GPIO** | 最多 40 个 | 注意复用优先级 |
| **定时器** | 内置通用定时器 | 做 PWM、延时、任务调度 |
| **温度传感器** | 内置 | 粗略温度监测 |
| **看门狗** | 独立 | 防死机复位 |

## 5. 软件开发生态

### 开发环境

| 工具 | 说明 |
|------|------|
| **MounRiver Studio (MRS)** | 沁恒官方免费 IDE，基于 Eclipse，内置 RISC-V GCC 工具链 |
| **WCH-Link** | 调试下载器（RISC-V 两线调试） |
| **ISP 工具** | 串口/USB 免调试器下载 |

### 软件框架

- **TMOS（Task Management Operating System）**
  - 沁恒提供的任务调度系统，非 RTOS，但支持任务分时和事件驱动
  - BLE 协议栈跑在 TMOS 上，用户应用建议也基于 TMOS 编写
  - 比裸机轮询更省功耗，比 FreeRTOS 更轻量

### 关键 SDK 资源

| 资源 | 位置/获取方式 |
|------|---------------|
| CH583 EVT 包 | 官网下载，含全部例程 + 蓝牙开发手册 |

| MESH 协议栈库 | `MESH/MESH_LIB/` 目录 |

## 6. Flash 与存储注意事项

⚠️ **这是踩坑重灾区，务必注意：**

### CodeFlash vs DataFlash

| 区域 | 基地址 | 最小擦除单位 | 特点 |
|------|--------|-------------|------|
| **CodeFlash** | `0x00000` | 4 KB（块擦） | 读写带内核加解密，擦后读前 4 字节为“乱码”属正常 |
| **DataFlash** | `0x70000` | 256 B（页擦）/ 4 KB（块擦） | 无加解密，偏移地址操作，可直接 ISP 读出 |

### 关键约束

1. **写前必擦**：Flash 不能直接覆盖写，必须先擦除
2. **Buff 地址 4 字节对齐**：RAM 中的读写缓冲区地址必须 4 字节对齐，否则可能 HardFault
3. **BLE 配对信息占用**：DataFlash 尾部 `0x7E00` ~ `0x7FFF` 用于 BLE 配对绑定，用户数据要避开
4. **OTA 标志位**：`0x7000` 处 4 字节用于 OTA 升级标志

### 实测时间开销（CH592 参考，CH58x 相近）

| 操作 | 数据量 | 耗时 |
|------|--------|------|
| 擦除 | 1 ~ 256 B | ~17 ms |
| 擦除 | 4096 B（1 块） | ~17 ms |
| 写入 | 1 B | ~2.2 ms |
| 写入 | 4096 B | ~25.5 ms |
| 读取 | 1 B | ~7 μs |

## 7. 硬件设计参考

### 最小系统要点

- 电源：2.3 ~ 3.6 V，建议 3.3 V LDO 供电
- 晶振：内置 HSI，外部 32.768 kHz 晶振（可选，用于 RTC 精度）
- 复位：外部 NRST 引脚 + 上电复位 POR
- 天线：PCB 板载天线或陶瓷天线（参考官方 SCHPCB 封装库）
- 调试：WCH-Link 两线调试，或串口 ISP 下载

### 引脚复用注意

CH582M 为 QFN48，很多引脚有 3 ~ 4 个复用功能，设计时要先查 **引脚定义表** 确认优先级。

| 维度 | 沁恒 CH582M | Nordic nRF52840 |
|------|-------------|-----------------|
